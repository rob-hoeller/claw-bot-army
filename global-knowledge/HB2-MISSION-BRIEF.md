## The Rules

1. **HB1 is the reference.** Every feature in HB1 must exist in HB2 before we ship. Use HB1 to test against.
2. **Ship through the pipeline.** No cowboy commits. Every feature: Spec → Design → Build → QA → Ship.
3. **Quality over speed.** We have a deadline but we don't ship broken code. IN6 has veto power.
4. **Communicate.** Use the team channels. Share blockers early. Ask for help. We're stronger together.
5. **Preserve what works.** If something in HB1 is great (like the agent detail panel with chat), carry it forward and make it better.
6. **New stack, new thinking.** Don't just port old code. Rethink patterns for TS/JS + Supabase. Build it right.
